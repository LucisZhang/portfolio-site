# Margin Control Tower

Status: dual-source, browser-native analytics case study. The fixed-seed synthetic fixture remains
the default guided walkthrough; a committed Olist-derived Parquet artifact supplies an explicit
real-data mode. If the real artifact is absent, malformed, or fails its browser contract, the UI
fails closed and keeps the synthetic fallback active.

## Decision workflow

A category manager selects a week, category, mapped region, and channel; reconciles the margin
bridge; compares cost drivers; and tests a bounded scenario without presenting that scenario as a
forecast or causal result.

## Synthetic fallback

`synthetic-margin-data.json` and CSV are generated by `npm run generate:analytics` with fixed seed
`2026071301`. They contain 9,360 fictional rows across 52 weeks, 20 products, five categories,
three regions, and three channels at `week x product x region x channel`. The final eight weeks are
a disjoint synthetic holdout. Every row is tagged `provenance=synthetic`; no external dataset
license or real commercial result is claimed for this fixture.

## Committed real artifact

Real mode loads [olist-margin.parquet](olist-margin.parquet), a 15,809-cell ZSTD aggregate derived
from the public Olist Brazilian e-commerce dataset: 99,441 source orders and 112,650 order-item
rows reconciled with customer state, payments, reviews, order status, product category, and
freight. Its grain is `week x product_category x mapped region x dominant payment channel`; it is
not product-level evidence.

- Authority: [Olist on Kaggle](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce)
- Retrieval date: 2026-07-17
- License: [CC BY-NC-SA 4.0](https://creativecommons.org/licenses/by-nc-sa/4.0/)
- Pipeline: [reproduction and locked inputs](../../../pipelines/olist-margin/README.md)
- Provenance: [source, transport, attribution, and output hashes](../../../pipelines/olist-margin/PROVENANCE.md)

Raw Olist rows are not committed. The browser downloads the Parquet once, computes SHA-256 over
those exact bytes, registers that same buffer with lazy DuckDB-WASM, and renders
[detection-report.json](detection-report.json) and
[elasticity-report.json](elasticity-report.json) only when each report names the identical hash.
The final eight observed weeks must be the only holdout weeks, with no week crossing splits.

## Measurement and proxy boundaries

- Item price and freight are observed after payment/review reconciliation.
- The discount proxy uses a shifted, expanding category-median reference price. A category's first
  observation has no prior history, so it explicitly falls back to current price and produces zero
  proxy discount for that cold-start observation.
- Returns are a disclosed order-status/review proxy; COGS is a disclosed 60% item-price proxy.
- Before STL, 95 observed totals are reindexed to the complete 106-Monday calendar; the 11 Mondays
  with no derived Olist cell are explicitly zero-filled. The detector evaluates six deterministic
  perturbations only on observed Mondays. Calendar-completion rows and replay labels never enter
  the Parquet, and no manually labeled real anomaly is claimed.
- The log-log price coefficient is fit on the analysis window. The later eight-week holdout is used
  only to evaluate MAPE; the estimate is associational, not causal.

The generated [methods evidence](methods-evidence.json) contains bilingual acquisition, cleaning,
split, modeling, label, metric, reproduction, and claim-boundary disclosures. This case study does
not claim audited company economics, causal lift, a forecast, or production decisioning.
