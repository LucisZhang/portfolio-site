# Margin Control Tower

Status: greenfield browser vertical slice built in 2026-07. It supersedes the inherited
revenue/RFM presentation as the core portfolio experience; the old Tableau link is prior work,
not evidence for this rebuild.

## Decision

A fictional category manager chooses which weekly category issue to fix and tests a bounded
promotion change before recording an action. The workflow is data -> contract -> metric semantics
-> diagnosis -> root cause -> scenario -> synthetic holdout check -> action.

## Data and license

`synthetic-margin-data.json` and CSV are generated by `npm run generate:analytics` with fixed seed
`2026071301`. It contains no external or inherited rows. The original synthetic fixture is covered
by the portfolio `NOTICE.md`; no external dataset license is asserted. Every row is tagged
`provenance=synthetic`.

The dataset contains 9,360 rows across 52 weeks, 20 products, five categories, three regions, and three channels. Grain: one row per `week x product x region x channel`; the final eight weeks are a disjoint synthetic holdout period. The data dictionary is the generated row schema plus
`data-contract.json`; metric formulas, owner, grain, provenance, unit, and decision use live in
`metric-registry.json`.

The same 9,360 rows are available as JSON, CSV, and real ZSTD-compressed Parquet. The Parquet
derivative is generated and row-count checked with `npm run generate:analytics-parquet` using the
locked DuckDB-Wasm dependency.

## Method and validation

- Ten deterministic data-quality checks fail closed.
- Contribution margin uses the registry formula and must satisfy its accounting identity.
- The selected anomaly is fixed-seed and explicitly injected; no detector-precision claim is made.
- The promotion slider uses a disclosed elasticity assumption, not a forecast.
- Scenario and synthetic holdout values are compared only as a demonstration of verification.

## Failure conditions and monitoring

The decision output is blocked on contract failure. A production version would monitor source
freshness, contract failures, metric-registry changes, anomaly volume, scenario error, and action
follow-through. This static slice has no warehouse, server, or sleeping external demo.

## Limitations

All business values are fictional synthetic currency units. The slice proves deterministic
analytics engineering and action design, not real commercial impact or causal lift.
