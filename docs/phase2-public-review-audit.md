# Phase 2 public Review audit

## Baseline

- Review URL: <https://portfolio-site-gpt-review.vercel.app>
- Captured: 2026-07-13 13:39 Asia/Shanghai (05:39 UTC)
- Browser: Google Chrome through Playwright, headless, fresh contexts, no stored cookies, service workers blocked
- Viewports: 1440 x 1000, 1024 x 900, 390 x 844
- Locales: English and Chinese
- Coverage: 11 routes, 66 route/locale/viewport states, 49 public files, 8 external links, 104 screenshots
- Raw report: [baseline/audit.json](phase2-public-review-artifacts/baseline/audit.json)

All 66 page requests and all 49 public-file requests returned HTTP 200. Page settlement ranged from 1.9 to 5.8 seconds. The audit found no horizontal document overflow and no application exception. Twenty-one files are linked from a case study, but every one currently opens as a raw response instead of in a portfolio viewer.

The external-link probe is not equivalent to a browser verdict. GitHub returned HTTP 429 for one otherwise valid `PUBLICATION.md` URL, and LinkedIn returned its non-standard HTTP 999 anti-bot response. These are recorded as manual-verification items, not as confirmed broken links.

## Confirmed baseline behavior

- Privacy text example, image box drawing, actual PNG download, and two-page PDF navigation completed without an interaction exception.
- p1 failure replay, Release Guardian approval/audit, RAG contract drift, Margin scenario, and Credit policy-decision interactions completed in English and Chinese at all three viewports where applicable.
- All 49 local public files returned a non-HTML content type matching their file class. The `.mmd` files are served as `application/vnd.chipnuts.karaoke-mmd` and need explicit viewer handling.
- No page had document-level horizontal overflow. This does not close A05: fitting a desktop tool into a narrow column is not the same as designing a usable mobile workflow.
- The only captured failed request was the Vercel Preview Toolbar script blocked by CSP.

## Required final pass

The same script must run against the new anonymous Review URL after implementation. A finding can be marked fixed only when the final report records its verification and, for visual findings, links to a final-state screenshot at the same viewport.
